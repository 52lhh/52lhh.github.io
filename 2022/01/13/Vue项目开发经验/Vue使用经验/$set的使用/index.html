Vue中set的使用

为什么要使用set

  1. 在vue中，并不是任何时候数据都是双向绑定的
  2. 官方文档中表示：由于JS的限制，Vue不能检查以下两种情况数组的变动：
    1. 当利用索引直接设置一个数组项时，如：vm.items[index] = newValue
    2. 当修改数组的长度时，如：vm.item.length = newLength
  3. 因此，以上两种情况就属于数据没有被双向绑定，需要使用$set

$set用法

  1. 解决数据没有被双向绑定我们可以使用vm.$set实例方法，该方法是全局方法Vue.set的一个别名

    // 数组
this.$set(原数组, 索引值, 需要赋的值) 

// 对象
this.$set(原对象，对象属性，需要赋的对象属性值)
// this.$set(this.bookInfo, 'canteenName', obj.dictLabel)

什么时候使用set

  1. set为解决双向绑定失效而生，只需要关注什么时候双向绑定失效就可以

  2. 由于Vue中只要值的地址没有改变，vue是检测不到数据变化的，针对这这一点，有以下三种情况

    1. 当利用索引直接设置一个数组项时，例如：vm.items[index] = newValue
    2. 当你修改数组的长度时，例如：vm.items.length = newLength
    3. 由于 JavaScript 的限制，Vue不能检测对象属性的添加或删除

  3. 针对第三点做如下说明：

    var vm = new Vue({
  data: {
    a: 1
  }
})
// vm.a现在是响应式的

vm.b = 2
// vm.b不是响应式的
    1. 从上面的代码可以知道vm.b不是响应式的，简单的来说，如果对象中原来没有这个key，新增的key是无法双向绑定的

  4. 这里我们同样可以用set去解决这个问题：

    // this.$set(原对象，对象属性，需要赋的对象属性值)
// vm.userProfile为Vue实例中的一个对象
// $set是全局方法 Vue.set
vm.$set(vm.userProfile, 'age', 27)
    1. 如果添加的属性很多条，可能就需要写一个循环来多次set

  5. 当然也可能使用Object.assign，这里有一些需要注意的地方，如果你想添加新的响应式属性，参见以下代码

    // 错误
Object.assign(vm.userProfile, {
  age: 27,
  favoriteColor: 'Vue Green'
})

// 正确
// Object.assign(原对象，对象属性，需要赋的对象属性值)
vm.userProfile = Object.assign({}, vm.userProfile, {
  age: 27,
  favoriteColor: 'Vue Green'
})
