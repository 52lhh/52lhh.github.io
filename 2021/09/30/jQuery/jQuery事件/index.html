jQuery事件注册

单个事件注册

  1. 语法：
    • $(“div”).click( function() { 事件处理程序 } )
  2. 其他事件和原生JS基本一致
    • 比如：mouseover、mouseout、blur、focus、changes、keydown、keyup、resize、scroll等

jQuery事件处理

事件处理on()绑定事件

  1. on()方法在匹配元素上绑定一个或多个事件的事件处理函数

  2. 语法

    • element.on(events, [selector], fn)

    1. events：一个或多个用空格分隔的事件类型，如”click或”keydown”

    2. selector：元素的子元素选择器

    3. fn：回调函数即绑定在元素身上的侦听函数

      // on()中逐个绑定事件
$(function () {
  $("div").on({
    mouseover: function () {
      $(this).css("backgroundColor", "pink");
    },
    mouseout: function () {
      $(this).css("backgroundColor", "skyblue");
    },
    click: function () {
      $(this).css("backgroundColor", "purple");
    }
  })

  // on()中同时绑定多个事件
  $("div").on("mouseenter mouseleave", function () {
    $(this).toggleClass("current");
  })
}

  3. 优势

    1. 可以绑定一个或者多个事件
    2. 可以事件委派操作；事件委派的定义就是把原来加给子元素身上的事件绑定在父元素身上，就是把事件委派给父元素
    3. 可以给未来动态创建的元素绑定事件
      $(function () {
  // 给未来动态创建的元素绑定事件
  $("ul").on("click", "li", function () {
    alert("22");
  })
  var li = $("<li>我是后来创建的</li>");
  $("ul").append(li); // 可以弹出22，用传统单个事件绑定则不能弹出
}

事件处理off()解绑事件

  1. off()方法可以移除通过on()方法添加的事件处理程序

  2. 解绑元素所有的事件处理程序

    • $(“div”).off();

  3. 解绑元素上面的一部分事件处理程序

    • $(“div”).off(“click”);

  4. 解绑事件委托

    • $(“ul”).off(“click”, “li”);

      <script>
  $(function () {
    $("div").on({
      click: function () {
        $(this).css("backgroundColor", "pink");
      },
      mouseover: function () {
        $(this).css("backgroundColor", "skyblue");
      }
    })

    $("ul").on("click", "li", function () {
      alert(11);
    })

    // 解除div的所有事件
    // $("div").off();

    // 解除div的click事件
    $("div").off("click");

    // 解除事件委托
    $("ul").off("click", "li");
  })
</script>

one()事件

  1. 只能触发事件一次
    $(function () {
  // one()事件
  $("div").one("mouseout", function () {
    $(this).css("backgroundColor", "black");
  })
}

自动触发事件trigger()

  1. 方式一：元素.事件(); — 会触发元素的默认行为
    • $(“div”).click();
  2. 方式二：元素.trigger(“事件”); — 会触发元素的默认行为
    • $(“div”).trigger(“focus”);
  3. 方式三：元素.triggerHandler(“事件”); — 不会触发元素的默认行为
    • $(“div”)..triggerHandler(“click”);

jQuery事件对象

  1. 事件被触发，就会有事件对象的产生
    • Element.on(evenets, [selector], function(event) {})
    1. 阻止默认行为：event.preventDefault() 或者 return false
    2. 阻止冒泡：event.stopPropagation()

jQuery其他方法导读

jQuery对象拷贝

  1. 若想要把某个对象拷贝（合并）给另外一个对象使用，此时可以使用$.extend()方法
  2. 语法
    • $.extend([deep], target, Object1, [objectN])
    1. deep：若设为true为深拷贝，默认为false浅拷贝
    2. target：要拷贝的目标对象
    3. object1：待拷贝到第一个对象的对象
  3. 浅拷贝和深拷贝
    1. 浅拷贝
      • 把原来对象里面的复杂数据类型的地址拷贝给目标对象
      • 会覆盖目标对象的里面的数据
    • 目标拷贝对象里面的值会随着原来对象的改变而改变，因为他们指向同一个地址
    1. 深拷贝
      • 把原来对象里面的数据完全复制一份给目标对象
      • 会覆盖目标对象的里面的数据
      • 若其中有不冲突的属性，会合并到一起，比如目标对象中的对象的属性age和原来对象中的对象的属性name
      • 目标拷贝对象里面的值不会随着原来对象的改变而改变

jQuery多库共存

  1. 问题概述：jQuery使用$作为标示符，随着jQuery的流行，其他js库也会用这$作为标识符，这样一起使用会引起冲突
  2. 客观需求：需要一个解决方案，让jQuery和其他的js库不存在冲突，可以同时存在，这就叫做多库共存
  3. 解决方案
    1. 如果$符号冲突，就使用jQuery代替$
    2. 让jquery释放对$控制权让用自己决定
      var suibian = jQuery.noConflict();
suibian("div");

toDoList案例

案例需求

  1. 文本框里面输入内容，按下回车，就可以生成待办事项
  2. 点击待办事项复选框，就可以把当前数据添加到已完成事项里面
  3. 点击已完成事项复选框，就可以把当前数据添加到待办事项里面
  4. 但是本页面内容刷新页面不会丢失

案例分析

  1. 刷新页面不会丢失数据，因此需要用到本地存储localStorage
  2. 核心思路：不管按下回车，还是点击复选框，都是把本地存储的数据加载到页面中，这样保证刷新关闭页面不会丢失数据
  3. 存储的数据格式：var todolist = [{ title: ‘xxx’, done: false}]
  4. 注意点1：本地存储localStorage里面只能存储字符串格式，因此需要把对象转换为字符串JSON.stringify(data)
  5. 注意点2：获取本地存储数据，需要把里面的字符串转换为对象格式JSON.parse()我们才能使用里面的数据

步骤一：toDoList按下回车把数据添加到本地存储里面

  1. 切记：页面中的数据，都要从本地存储里面获取，这样刷新页面不会丢失数据，所以先要把数据保存到本地存储里面
  2. 利用事件对象.keyCode判断用户按下回车键(回车键的ASCII码值为13)
  3. 声明一个数组，保存数据
  4. 先要读取本地存储原来的数据(声明函数getData())，放到这个数组里面
  5. 之后把最新从表单获取过来的数据，追加到数组里面
  6. 最后把数组存储给本地存储(声明函数savaData())

步骤二：toDoList本地存储数据渲染加载到页面

  1. 因为后面也会经常渲染加载操作，所以声明一个函数load，方便后面调用
  2. 先要读取本地存储数据(数据不要忘记转换为对象格式)
  3. 之后遍历这个数据($.each())，有几条数据，就生成几个小li添加到ul里面
  4. 每次渲染前，先把原先里面ul的内容清空，然后渲染加载最新的数据

步骤三：toDoList删除操作

  1. 点击里面的a链接，不是删除的li，而是删除本地存储对应的数据
  2. 核心原理：先获取本地存储数据，删除对应的数据，保存给本地存储，重新渲染列表li
  3. 我们可以给链接自定义属性记录当前的索引号
  4. 根据这个索引号删除相关的数据 — 数组的splice(i, 1)删除元素方法
  5. 存储修改后的数据，然后存储给本地存储
  6. 重新渲染加载数据列表
  7. 因为a是动态创建的，使用on方法绑定事件

步骤四：toDoList正在进行和已完成选项操作

  1. 当点击了小的复选框，修改本地存储数据，再重新渲染数据列表
  2. 点击之后，获取本地存储数据。
  3. 修改对应数据属性done为当前复选框的checked状态
  4. 之后保存数据到本地存储
  5. 重新渲染加载数据列表
  6. load加载函数里面，新增一个条件，如果当前数据的done为true就是已经完成的，就把列表渲染加载到ul里面
  7. 如果当前数据的done为false，则是待办事项，就把列表渲染加载到ol里面

步骤五：toDoList统计正在进行个数和已经完成个数

  1. 在我们load函数里面操作
  2. 声明2个变量：todoCount待办个数doneCount已完成个数
  3. 当进行遍历本地存储数据的时候，如果数据done为false，则todoCount++，否则doneCount++
  4. 最后修改相应的元素text()
