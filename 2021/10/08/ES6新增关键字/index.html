ES6新增关键字

发表于 2021-10-08 分类于 ES6

学习笔记：ES6新增关键字
个人笔记可能会存在一些错误，您在阅读中如若发现，欢迎您的指正

ES6

新增关键字

let变量

  1. let变量：ES6中新增的用于变量声明的关键字

    1. 例：
      let a;
let b, c, d;
let e =100;
let f = 521, g = 'love', h = [];

  2. 特性：

    1. 变量不能重复声明
    2. 块级作用域，只在所处代码块内部有效
      • 在一个大括号中，使用let关键字声明的变量才具有块级作用域，var关键字不具备这个特点
    3. 防止循环变量变成全局变量
    4. 不存在变量提升
    5. 不影响作用域链的效果

  3. 经典面试题

    <script>
  var arr = [];
  for (var i = 0; i < 2; i++) {
    arr[i] = function () {
      console.log(i);
    }
  }
  // 因为变量i是全局变量，函数执行时输出的是全局作用域的i值
  // 而for循环是一个同步任务，函数执行时，循环已经结束，因此i值为2
  arr[0](); // 2
  arr[1](); // 2

  let array = [];
  for (let i = 0; i < 2; i++) {
    array[i] = function () {
      console.log(i);
    }
  }
  // 每次循环都会产生一个块级作用域，每个块级作用域中的i值都是不同的
  // 函数执行时，输出的是自己上一级（循环产生的块级作用域）中的i值
  array[0](); // 0
  array[1](); // 1
</script>

const关键字

  1. const关键字：用来声明常量，常量就是值（内存地址）不能变化的量
  2. 特性：
    1. 用const声明常量一定要赋初始值
    2. 一般常量使用大写（潜规则）
    3. 块级作用域，只在所处代码块内部有效
    4. 常量赋值后，值不能修改，实际上是值对应的内存地址不可更改
      • 基本数据类型：常量声明后，值不可更改
      • 复杂数据类型（数组对象）：数据结构内部的值可以更改，数据本身不可更改
    5. 对于数组和对象进行元素修改，不算做是对常量的修改，不会报错；但是如果对数组和对象的值直接进行修改，则会报错，因为改变内存地址；因此声明数组和对象时推荐使用const关键字
      <script>
  const ARR = [1, 'a', 2, 'b'];
  ARR.push(3); // 不报错，对数组和对象进行元素修改
  ARR = [1, 3, 5]; // 报错，对数组和对象的值直接进行修改，改变了地址
  ARR = 100; // 报错，对数组和对象的值直接进行修改，改变了地址
</script>

let、const、var声明变量的区别

  1. 使用var声明的变量，其作用域为该语句所在的函数内，且存在变量提升现象
  2. 使用let声明的变量，其作用域为该语句所在的代码块内，不存在变量提升
  3. 使用const声明的是常量，在后面出现的代码中不能再修改该常量的值
    var let const
    函数级作用域 块级作用域 块级作用域
    变量提升 不存在变量提升 不存在变量提升
    值可改变 值可改变 值不可改变
