学习笔记：Vue-组件化及模块化

组件化

组件化思想

  1. 如果将一个页面中所有的处理逻辑全部放在一起，处理起来就会变得非常复杂，而且不利于后续的管理以及扩展
  2. 但如果，将一个页面拆分成一个个小的功能块，每个功能块完成属于自己这部分独立的功能，那么之后整个页面的管理和维护就变得非常容易了

Vue组件化思想

  1. 它提供了一种抽象，让我们可以开发出一个个独立可复用的小组件来构造我们的应用
  2. 任何的应用都会被抽象成一颗组件树
  3. 组件化思想应用
    1. 有了组件化的思想，我们在之后的开发中就要充分的利用它
    2. 尽可能的将页面拆分成一个个小的、可复用的组件
    3. 这样让我们的代码更加方便组织和管理，并且扩展性也更强

注册组件的基本步骤

  1. 组件使用的三个步骤
    1. 创建组件构造器：调用Vue.extend()方法创建组件构造器
      • Vue.extend()：传递一个对象作为参数，参数对象中有一个属性template — (模板)
    2. 注册组件：调用Vue.component()方法注册组件 — component(组件）
      • Vue.component(tagName, options)
        • tagName：为组件名
        • options：为配置选项（组件）
    3. 使用组件：在Vue实例的作用范围内使用组件 ## 组件化基本使用 ### 组件化的基本使用过程
  2. 创建组件构造器对象 — Vue.extend()
    1. 调用Vue.extend()创建的是一个组件构造器
    2. 通常在创建组件构造器时，传入template代表我们自定义组件的模板
    3. 该模板就是在使用到组件的地方，要显示的HTML代码
    4. 事实上，这种写法在Vue2.x的文档中几乎已经看不到了，它会直接使用下面我们会讲到的语法糖，但是在很多资料还是会提到这种方式，而且这种方式是学习后面方式的基础
  3. 注册组件 — component()
    1. 调用Vue.component()是将刚才的组件构造器注册为一个组件，并且给它起一个组件的标签名称
    2. 所以需要传递两个参数: 1、注册组件的标签名 2、组件构造器
  4. 使用组件
    1. 组件必须挂载在某个Vue实例下，否则它不会生效

全局组件和局部组件

  1. 全局组件：意味着可以在多个Vue的实例下使用
  2. 局部组件：在某一个Vue的实例下注册的组件，只能在该实例下使用

父组件和子组件

  1. 组件和组件之间存在层级关系
  2. 其中非常重要的关系就是父子组件的关系

注册组件语法糖

  1. Vue为了简化注册组件的这个过程，提供了注册的语法糖
  2. 主要是省去了调用Vue.extend()的步骤，而是可以直接使用一个对象来代替
  3. 语法糖注册全局组件和局部组件

组件的分离写法

  1. 通过语法糖简化了Vue组件的注册过程，另外还有一个地方的写法比较麻烦，就是template模块
  2. 如果我们能将其中的HTML分离出来写，然后挂载到对应的组件上，结构会变得非常清晰
  3. Vue提供了两种方案来定义HTML模块内容：
    1. 使用<script>标签，type值为text/x-template
    2. 使用<template>标签，添加id值，用于注册组件时进行挂载到template上

为什么组件的data必须是函数

  1. 组件是一个单独功能模块的封装
  2. 这个模块有属于自己的HTML模板，也应该有属性自己的数据data
  3. 每个组件的内部应该有自己存储数据的地方
    1. 组件对象也有一个data属性（也可以有methods等属性）
    2. 只是这个data属性必须是一个函数
    3. 而且这个函数返回一个对象，对象内部保存着数据
  4. 组件就是指向Vue的，组件的原型就是Vue
  5. 为什么组件的data必须是函数（面试重点）
    1. 在实际开发中，组件是需要进行复用的，每个组件都有自己的逻辑，会避免产生连锁反应
    2. 当组件的data为对象时，当需要多次使用组件时，就会让组件的多个实例都执行data，因为data为对象，则多个实例都会调用这个对象，从而导致这些实例的data指向相同，产生连锁反应，使改变一个组件数据，导致多个组件的内容同时改变
    3. 而当组件的data为函数时，当有多个实例时，每个实例的data都包含在闭包之中，则不会互相干扰 ## 父子组件的通信
  6. 我们提到了子组件是不能引用父组件或者Vue实例的数据的
  7. 但是，在开发中，往往一些数据确实需要从上层传递到下层：
    1. 比如在一个页面中，我们从服务器请求到了很多的数据
    2. 其中一部分数据，并非是我们整个页面的大组件来展示的，而是需要下面的子组件进行展示
    3. 这个时候，并不会让子组件再次发送一个网络请求，而是直接让大组件(父组件)将数据传递给小组件(子组件)
  8. 如何进行父子组件间的通信，有两种方式
    1. 通过props向子组件传递数据（父传子）
    2. 通过自定义事件向父组件发送消息 （子传父）
  9. 真实的开发中，Vue实例和子组件的通信和父组件和子组件的通信过程是一样的

父传子使用props

  1. 通过props向子组件传递数组类型数据
  2. props数据验证
    1. props数据可以支持许多种数据类型
      • String — 字符型
      • Number — 数值型
      • Boolean — 布尔型
      • Array — 数组型
      • Object — 对象
      • Date — 日期
      • Function — 函数
      • Symbol —
    2. 当有自定义构造函数时，验证也支持自定义的类型

子传父使用自定义事件

  1. 子组件传递数据或事件到父组件中，需要用到自定义事件来完成
  2. 什么时候需要使用自定义事件？
    1. 当子组件需要向父组件传递数据时，就要用到自定义事件了
    2. 之前学习的v-on不仅仅可以用于监听DOM事件，也可以用于组件间的自定义事件
  3. 自定义事件流程
    1. 在子组件中，通过$emit(‘自定义事件名’, 传递的数据)来触发事件，向父组件发射数据
    2. 在父组件中，通过v-on/@来监听子组件自定义事件
  4. 案例：子传父计数器

父子组件的访问方式

父访问子：$children和$refs

  1. 有时候需要父组件直接访问子组件，子组件直接访问父组件，或者是子组件访问根组件
    1. 父组件访问子组件：使用$children或$refs
      • $refs更为常用，百分之九十的情况使用$refs
      • $children是根据索引进行获取子组件，当子组件的顺序发生变化时，则使用此方法不能达到目的，因此一般只有需要获取所有组件时，才会使用$children，返回值是包含所有子组件的数组 ### 子访问父：$perent和$root
  2. $perent：用于访问父组件，实际开发中基本不使用
    1. 原因：因为组件需要保持自身的复用性，要降低和其他组件之间的耦合度，如果使用了$perent就会使子组件与父组件之间的耦合度变大，降低了复用性
  3. $root：用于访问根组件，实际开发中也很少使用
    1. 原因：因为根组件中很少会有具有实际意义的属性和方法，因此基本不会访问根组件

组件化高级 - 插槽

为什么使用slot（插槽）

  1. 组件的插槽
    1. 组件的插槽也是为了让我们封装的组件更加具有扩展性
    2. 让使用者可以决定组件内部的一些内容到底展示什么
  2. 比如：移动网站中的导航栏
    1. 移动开发中，几乎每个页面都有导航栏
    2. 导航栏我们必然会封装成一个插件，比如nav-bar组件
    3. 一旦有了这个组件，我们就可以在多个页面中复用了
  3. 但是由于这类组件它们的大体是相同的，但是仍然有部分内容是不同的，因此不可能每一个都单独封装为一个组件，也不可能将这一类型的模块全部封装为一个组件
  4. 因此进行封装的时候就要：抽取共性，保留不同
    1. 最好的封装方式就是将共性抽取到组件中，将不同暴露为插槽
    2. 一旦我们预留了插槽，就可以让使用者根据自己的需求，决定插槽中插入什么内容
    3. 是搜索框，是文字，还是菜单，由调用者自己来决定

插槽的基本使用

  1. 在组件的template模板中，将组件需要进行替换的部分用插槽标签代替；在使用组件时，在定义的组件标签内写上需要替换的标签元素即可
  2. 可以给插槽设置默认值，这样使用组件时，没有定义插槽元素的组件标签则使用插槽的默认元素
  3. 如果有多个值放到组件标签中，则插槽会将所有元素都进行替换

具名插槽

  1. 当子组件的功能复杂时，子组件的插槽可能并非是一 个；比如我们封装一个导航栏的子组件，可能就需要三个插槽，分别代表左边、中间、右边，这个时候就需要给插槽取名字，这就是具名插槽
  2. 具名插槽会根据插槽名让指定插槽修改元素
  3. 若组件标签添加元素时，没有指定插槽名，则会修改所有没有命名的插槽

编译的作用域

  1. 父组件模板的所有东西都会在父级作用域内编译；子组件模板的所有东西都会在子级作用域内编译，不会存在跨域的现象

作用域插槽的使用

  1. 作用域插槽：父组件替换插槽的标签，但是数据内容由子组件来提供
  2. 作用域插槽使用：
    1. 当数据在子组件中，而需要将数据在父组件进行展示，由于编译的作用域，父组件是不可以在子组件中获取数据的
