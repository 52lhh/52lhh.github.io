学习笔记：继承
个人笔记可能会存在一些错误，您在阅读中如若发现，欢迎您的指正

继承

  1. ES6之前并没有给我们提供extends继承。我们可以通过构造函数+原型对象模拟实现继承，被称为组合继承

改变this指向的三种方法

call()

  1. 调用这个函数，并且修改函数运行时的this指向

  2. call()的两个作用

    1. call()：可以调用函数

    2. 可以改变这个函数的this指向

      <script>
  function fn(x, y) {
    console.log('hello world!');
    console.log(this);
  }
  // 调用函数
  fn.call();

  var man = {
    name: '男人'
  }
  // 改变函数的指向
  fn(); // 指向Window
  // 第一个参数改变函数指向，后面的参数作为形参
  fn.call(man, 1, 2) // 指向对象man
</script>

apply()

  1. apply()的两个作用
    1. apply()：可以调用函数
    2. 可以改变这个函数的this指向
  2. 与call()的不同在于，apply()后面传递的参数必须是数组形式
  3. 因此apply()的主要应用于数组

bind()（重点）

  1. 该方法经常使用
  2. bind()可以改变这个函数的this指向
  3. bind()不会调用原来的函数
  4. 返回的是原函数改变this之后产生的新函数
  5. 当有些函数不需要立即调用，但是又需要改变这个函数内部的this指向时使用bind()
  6. 例：点击按钮后禁用3秒、tab栏切换案例运用（代码实现见vscode）
    <body>
  <button>按钮</button>
  <button>按钮</button>
  <button>按钮</button>
  <script>
    var btns = document.querySelectorAll('button');
    console.log(btns);
    for (var i = 0; i < btns.length; i++) {
      btns[i].addEventListener('click', function () {
        // 当disabled属性值为true时，让按钮处于不可点击状态
        this.disabled = true;
        setTimeout(function () {
          // 当disabled属性值为false时，让按钮处于不可点击状态
          // 这个this原本随着定时器指向Window，因为bind()方法改变了定时器的指向，所以此时this指向当前被点击的按钮
          this.disabled = false;
          // 让定时器调用bind()方法，因为bind()方法处于定时器外部，因此bind(this)中的this指向当前被点击的按钮
          // 定时器原来是指向Window，现在利用bind()方法指向了当前被点击的按钮，且不调用，随着定时器的执行而调用
        }.bind(this), 1000)
    })
    }
  </script>
</body>

call()、apply()、bind()总结

  1. 相同点：都可以改变函数内部的this指向
  2. 区别点：
    1. call和apply会调用函数，并且改变函数内部this指向
    2. call和apply传递的参数不一样，call传递参数aru1，aru2..形式；apply必须是数组形式[arg]
    3. bind不会调用函数，可以改变函数内部this指向
  3. 主要应用场景
    1. call经常做继承
    2. apply经常跟数组有关系，比如借助于数学对象实现数组最大值最小值
    3. bind不调用函数，但是还想改变this指向，比如改变定时器内部的this指向

借用父构造函数继承属性和方法

借用父构造函数继承属性

  1. 核心原理：通过call()把父类型的this指向子类型的this，这样就可以实现子类型继承父类型的属性
    <script>
  function Father(uname, age) {
    this.uname = uname;
    this.age = age;
  }
  function son(uname, age, sex) {
    // 借用父构造函数继承属性
    Father.call(this, uname, age);
    this.sex = sex;
  }
  var son = new son('张三', 15, '女');
  console.log(son);
</script>

借用父构造函数继承方法

  1. 因为构造函数的方法会放在原型对象中，因此父构造函数的方法放在父原型对象内部，若子构造函数的实例对象要想继承使用父原型对象内部的方法，则进行如下过程

    1. 将父构造函数实例化，让子原型对象指向父实例化对象，因为父实例化对象的原型（__proto__）为父原型对象，因此父实例化对象可以使用父原型对象中的方法，从而让指向父实例对象的子原型对象继承到父原型对象中的方法
    2. 注意：让子原型对象指向父实例对象后，会让父实例对象覆盖子原型对象，此时需要利用constructor指回子构造函数

  2. 注意：千万不可直接将父原型对象直接赋值给子原型对象，这样会让子原型对象直接指向父原型对象，导致一旦给子原型对象添加方法，会让父原型对象同时添加这个方法

    <body>
  <script>
    function Father(uname, age) {
      this.uname = uname;
      this.age = age;
    }
    function Son(uname, age, sex) {
      // 借用父构造函数继承属性
      Father.call(this, uname, age);
      this.sex = sex;
    }

    // 在父原型对象中添加方法
    Father.prototype.money = function () {
      console.log('赚钱赚钱');
    }

    // 千万不可直接将父原型对象直接赋值给子原型对象，这样会让子原型对象直接指向父原型对象，导致一旦给子原型对象添加方法，会让父原型对象同时添加这个方法
    Son.prototype = Father.prototype;

    // 借用父构造函数继承方法
    // 父构造函数实例化，让子原型对象指向父实例化对象
    Son.prototype = new Father();
    // 因为new Father()创建了一个对象，覆盖了子原型对象，因此需要让子原型对象手动指回构造函数
    Son.prototype.constructor = Son;

    // 子原型对象成功继承父原型对象中的方法
    Son.prototype.money(); // 输出：赚钱赚钱
    console.log(Son.prototype); // 子原型对象指向父实例对象
    console.log(Son.prototype.constructor); // 指向子构造函数

  </script>
</body>
