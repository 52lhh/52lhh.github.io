项目开发

项目目录结构

src文件下的目录划分

  1. assets资源文件夹：该文件夹下一般需要创建两个文件夹
    1. img文件夹
    2. css文件夹
  2. components组件文件夹：该文件夹下一般创建两个文件夹
    1. common文件夹：在内部放置的是公共的组件，在开发其他项目时可以直接进行使用的
    2. content文件夹：在内部放置的是当前开发项目中公共的组件，在开发其他项目时不适合用
  3. views视图文件夹：该文件夹下放置当前开发项目主要视图的文件夹，比如主页，注册页
  4. router路由文件夹：放置路由相关文件
  5. store状态管理文件夹
  6. network文件夹：放置网络相关的文件
  7. common公共文件夹：放置公共的js文件

配置文件

  1. css文件：在assets资源文件夹下的css文件夹中配置normalize.css文件和base.css文件

  2. 别名文件：创建vue.config.js文件，在其中配置各文件夹的别名

  3. 将vue2中的static文件夹下的editorconfig文件复制到当前vue3项目的src文件夹中

  4. 项目的模块划分：tabber -> 路由映射关系

项目优化

项目优化策略

生成打包报告

  1. 打包时，为了直观地发现项目中存在的问题，可以在打包时生成报告。生成报告的方式有两种：

    1. 通过命令行参数的形式生成报告

      vue-cli-service build --report

    2. –report选项可以生成report.html以帮助分析包内容

  2. 通过可视化的UI面板（命令语句：vue ui）直接查看报告（推荐）

    1. 在可视化UI面板中，通过控制台和分析面板，可以方便看到项目中所存在的问题

通过vue.config.js修改webpack的默认配置

  1. 通过vue-cli3.0工具生成的项目，默认隐藏了所有webpack的配置项，目的是为了屏蔽项目的配置过程，让程序员把工作的重心，放到具体功能和业务逻辑的实现上
  2. 如果程序员有修改webpack默认配置的需求，可以在项目根目录中，按需创建vue.config.js这个配置文件，从而对项目的打包发布过程做自定义的配置（具体配置参考 https://cli.vuejs.org/zh/config/#vue-config-js）

为开发模式与发布模式指定不同的打包入口

  1. 默认情况下，Vue项目的开发模式与发布模式，共用同一个打包的入口文件(即src/mainjs)。为了将项目的开发过程与发布过程分离，我们可以为两种模式，各自指定打包的入口文件，即:
    1. 开发模式的入口文件为：src/main-dev.js
    2. 发布模式的入口文件为：src/main-prodjs
  2. configureWebpack和chainWebpack
    1. 在vue.configjs导出的配置对象中，新增configureWebpack或chainWebpack节点，来自定义webpack的打包配置
    2. configureWebpack和chainWebpack的作用相同，唯一的区别就是它们修改webpack配置的方式不同：
      • chainWebpack 通过链式编程的形式，来修改默认的webpack配置
      • configureWebpack 通过操作对象的形式，来修改默认的webpack配置

通过externals加载外部CDN资源

  1. 默认情况下，通过import语法导入的第三方依赖包，最终会被打包合并到同一个文件中，从而导致打包成功后，单文件体积过大的问题

  2. 为了解决上述问题，可以通过webpack的externals节点，来配置并加载外部的CDN资源；凡是声明在externals中的第三方依赖包，都不会被打包；

  3. 实现原理：将项目中的依赖项声明到externals节点中，在打包期间，Webpack会检查程序员在externals节点中是否声明第三方依赖包，若有声明，则打包的时候Webpack就不会将相应的第三方依赖项进行打包，而当项目需要用到这些依赖时会直接到Window全局对象中寻找使用

通过CDN优化ElementUI的打包

  1. 虽然在开发阶段，我们启用了element-ui组件的按需加载，尽可能的减少了打包的体积，但是那些被按需加载的组件，还是占用了较大的文件体积。此时，我们可以将element-ui中的组件，也通过CDN的形式来加载，这样能够进一步减小打包后的文件体积

  2. 使用方法

    1. 在main-prod.js中，注释掉导入及使用Elemen-uit和element-ui按需加载的全部代码

    2. 在index.html的头部区域中，通过CDN加载element-ui的js和css样式

首页内容定制

  1. 不同的打包环境下，首页内容可能会有所不同。我们可以通过插件的方式进行定制，插件配置如下：

    chainWebpack: config => {
    // 发布模式
    config.when(process.env.NODE_ENV === 'production', config => {
        // 首页内容定制，不同的打包环境下，首页内容可能会有所不同
        config.plugin('html').tap(args => {
            args[0].isProd = true
            return args
        })
    })
    
    // 开发模式
    config.when(process.env.NODE_ENV === 'development', config => {
        // 首页内容定制，不同的打包环境下，首页内容可能会有所不同
        config.plugin('html').tap(args => {
            args[0].isProd = false
            return args
        })
    })
}

  2. 在public/index.html首页中，可以根据isProd的值，来决定如何渲染页面结构：

项目上线

项目上线相关配置

通过node创建web服务器

  1. 创建node项目，并安装express，通过express快速创建web服务器，将vue打包生成的dist文件夹，托管为静态资源即可

  2. 使用方法

    1. 创建一个新的空文件夹，命名：项目文件夹名_server

    2. 打开文件夹对应路径的终端，初始化包管理配置文件：npm init -y

    3. 安装express：npm i express -S

    4. 将项目文件夹中的vue打包生成的dist文件夹，复制到“项目文件夹名_server”文件夹中

    5. 在“项目文件夹名_server”文件夹中新建入口文件：app.js文件，书写以下代码

      // 导入express
const express = require("express")
// 创建web服务器
const app = express()
// 托管静态资源
app.use(express.static("./dist"))
// 启动web服务器
app.listen(80, () => {
    console.log("web server running at http://127.0.0.1");
})

    6. 终端输入：node .\app.js，打开http://127.0.0.1，在服务器运行项目

开启gzip压缩

  1. 使用gzip可以减小文件体积，使传输速度更快

    1. 可以通过服务器端使用Express做gzip压缩

  2. 使用方法

    1. 安装相应包：npm install compression -S

    2. 导入包：const compression = require(“compression”);

    3. 启用中间件：app.use(compression());

      • 注意：此代码必须写到静态资源托管之前

配置HTTPS服务

  1. 为什么要启用HTTPS服务

    1. 传统的HTTP协议传输的数据都是明文，不安全
    2. 采用HTTPS协议对传输的数据进行了加密处理，可以防止数据被中间人窃取，使用更安全

  2. 申请SSL证书 – 了解即可，一般由后端进行（ https://freessl.org)

    1. 进入https://freessl.cn/官网，输入要申请的域名并选择品牌
    2. 输入自己的邮箱并选择相关选项
    3. 验证DNS (在域名管理后台添加TXT记录)
    4. 验证通过之后，下载SSL证书(full_chain.pem公钥; private.key私钥)

  3. 在后台项目中导入证书

    1. 将申请到的SSL证书，放到项目根目录

    2. 在app.js文件导入https和fs模块

    3. 创建options配置对象

    4. 创建https类型的server服务器

使用pm2管理应用

  1. 在服务器中安装pm2：npm i pm -g
  2. 启动项目：pm2 start 脚本 –name 自定义名称
  3. 查看运行项目：pm2 ls
  4. 重启项目：pm2 restart 自定义名称
  5. 停止项目：pm2 stop 自定义名称
  6. 删除项目：pm2 delete 自定义名称
