学习笔记：Vuex
个人笔记可能会存在一些错误，您在阅读中如若发现，欢迎您的指正

Vuex

  1. 官方解释：Vuex是一个专为Vue.js应用程序开发的状态管理模式
    1. 他采用集中式存储管理应用的所有组件的状态，并一相应的规则保证状态以一种可预测的方式发生变化
  2. 状态管理是什么？
    1. 可以简单的将其看成把需要多个组件共享的变量全部存储在一个对象里面
    2. 然后，将这个对象放在顶层的Vue实例中，让其他组件可以使用
  3. Vuex就是为了提供这样一个在多个组件间共享状态的插件

状态管理

单界面的状态管理

  1. 单页面状态：State -> View -> Actions -> State

  2. 单页面转台管理：

    1. State：状态
    2. View：视图层，针对State的变化，显示不同的信息
    3. Action：这里的Action主要是用户的各种操作：点击、输入等等，会导致State状态的改变

多界面状态管理切换

  1. Vue已经做好了单界面的状态管理，对于多状态的界面管理

    1. 多个视图都依赖同一个状态
    2. 不同界面的Action都想修改同一个状态

  2. Vuex —全局单例模式（大管家）

    1. vuex基本思想：将共享的状态抽取出来，交给大管家，统一进行管理，之后每个视图，都按照规定好的进行访问和修改等操作

  3. 使用Vuex

    1. 安装vuex：npm install vuex –save

    2. 提取出一个公共的store对象，用于保存在多个组件中共享的状态

    3. 将store对象放置在new Vue对象中，这样可以保证在所有的组件中都可以使用到

    4. 在其他组件中使用store对象中保存的状态即可

      • 通过this.$store.state属性的方式来访问状态
      • 通过this.$store.commit(“decrement”)的方法来修改状态

  4. 注意：

    1. 修改State状态时，要通过mutations的方式改变，不能直接通过store.state.count改变

    2. 原因：Vuex可以通过devtools明确的追踪state状态的变化，所以不要直接改变store.state.count的值

Vuex核心概念

state单一状态树

  1. 也叫做单一数据源
  2. 单一状态树就是指：在开发中，推荐将所有的state状态信息保存在一个store对象中

getters

  1. 类似于单个组件中的computed计算属性

  2. 当需要从store中获取一些state变异后的状态，就可以使用Getter

  3. grtters作为参数和传递参数

    1. getter默认是不能传递参数的，如果希望传递参数，那么只能让getters本身返回另一个函数

Mutations

  1. Vuex的store中state状态的更新唯一方式就是通过：提交Mutations

  2. mutations的携带参数传递

  3. mutation同步函数：通常情况下，Vuex要去我们Mutation中的方法必须是同步方法

    1. 主要的原因是当我们使用devtools时，可以devtools可以帮助我们捕捉mutation的状态
    2. 但是如果是异步操作，那么devtools将不能很好的追逐这个操作什么时候会被完成
    3. 所以在mutation中不要定义异步操作，将异步操作放入Action中

Actions

  1. Actions类似于Mutation，但是是用来代替Mutation进行异步操作的
  2. 在Actions中定义的函数，默认属性为context（上下文）

Modules

  1. modles是模块的意思，为什么需要在Vuex中使用模块

    1. Vue使用单一状态树，那么也就意味着很多状态都会交给Vuex来管理
    2. 当应用变得非常复杂时，store对象就有可能变得相当臃肿
    3. 为了解决这个问题，Vuex允许我们将store分割成模块（module），而每个模块都拥有自己的state、mutations、actions、getters等

  2. modles模块中state、mutations、actions、getters的使用

    1. state：使用与store中的state相同，但在访问模块中的state时与store中的state有所不同

    2. mutations：使用与store中的mutations相同，但注意在mutations中的函数名与store中的mutations不能相同

    3. getters：使用与store中的getters相同

    4. actions

数据响应式原理

  1. 原理：在store的state中定义的属性都会被加入到响应式系统中，而响应式系统会监听属性的变化，当属性发生变化时，会通知所有界面中用到该属性的地方，让界面进行刷新

  2. 要想让state的数据为响应式，就必须遵守Vuex对应的规则

    1. 提前在store中初始化好所需的属性
    2. 当给state中的对象添加新属性时，使用下面的方式
      • 方式一：使用Vue.set(obj, ‘newProp’, 123)
      • 方式二：用新对象给就旧对象赋值
