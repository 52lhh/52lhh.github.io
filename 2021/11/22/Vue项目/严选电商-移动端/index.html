严选电商-移动端

项目结构分析

  1. build：webpack相关的配置文件夹（基本不需要更改）
  2. config：webpack相关的配置文件夹（基本不需要更改）
    • index.js：指定后台服务的端口号和静态资源文件夹
  3. node_modules
  4. src：源码文件夹
    • main.js：应用入口js
  5. static：静态资源文件夹
  6. .babelrc：babel的配置文件
  7. .editorconfig：通过编辑器的编码/格式进行一定的配置
  8. .eslintignore：eslint检查忽略的配置
  9. .eslintrc.js：eslint检查的配置
  10. .gitignore：git版本管制忽略的配置
  11. index.html：主页面文件
  12. package.json：应用包配置文件
  13. README.md应用描述说明的readme文件

项目源码目录设计

  1. src文件夹下的文件目录为
    1. api文件夹：与后台交互模块文件夹
    2. common文件夹：通用资源文件夹，如fonts、img
    3. components文件夹：非路由组件文件夹
    4. filters文件夹：自定义过滤器模块文件夹
    5. mock文件夹：模拟数据接口文件夹
    6. pages文件夹：路由组件文件夹
    7. router文件夹：路由器文件夹
    8. store文件夹：vuex相关模块文件夹
    9. App.vue：应用组件
    10. main.js：入口js

axios

  1. axios是在Vue和React等大型项目中，常用的数据请求方式。axios 是一个基于 promise 的 HTTP 库，可以用在浏览器和 node.js 中

  2. 官网地址：http://www.axios-js.com/

  3. 安装

    npm install axios

  4. axios统一使用格式

    axios.get(url[, config])
axios.post(url[, data[, config]])

// post请求
axios.post(url, {
    params: {}
  })
  .then(res=>{})
  .catch(err=>{})

// get请求
axios.get('/user')
  .then(res=>{})
  .catch(err=>{})

qs

  1. qs能将json格式转为 key=value 的格式

  2. 具体用法：

    const qs = require('qs');
axios.post('/foo', qs.stringify({ 'bar': 123 }));

  3. qs.stringify与JSON.stringify的区别

    1. qs.stringify()将对象序列化成URL的形式，以&进行拼接；JSON.stringify()是正常类型的JSON

      var a = {name:'Lucy',age:10};
qs.stringify(a)
// 'name=Lucy&age=10'
JSON.stringify(a)
// '{"name":"Lucy","age":10}'

开发项目中的小问题

开启后端服务

  1. 使用node运行接口文件，在cmd窗口中运行node 文件名（如：server.js），到浏览器输入 http://localhost:8000 查看接口文档

组件中发起请求

  1. 发送get请求：

    // get请求
axios.get("http://localhost:8000/getdata").then(res => {
  console.log(res);
});

  2. 发送post请求

    // qs.stringify()将对象序列化成URL的形式，以&进行拼接
let qsObj = qs.stringify({
  num: 123
});
// post请求
axios.post("http://localhost:8000/postdata", {
    data: qsObj
  })
  .then(res => {
    console.log(res);
  });

解决跨域

  1. 发现跨域问题，解决方案是找到 vue.config.js :

    module.exports = {
    devServer: {
        proxy: "http://localhost:8000"
    }
}

重置CSS样式

  1. 开发前需要将默认样式进行重置，使用reset-css

  2. 安装

    npm i reset-css

  3. 注意：若使用reset-css导致影响组件库的使用（如：vant），则放弃使用reset-css

  4. 使用reset-css会修改定义的html的字体大小，若要避免这个问题，则需要修改源码

    1. 打开node_modules文件夹

    2. 找到reset-css文件夹

    3. 打开reset.css文件，将html删除
